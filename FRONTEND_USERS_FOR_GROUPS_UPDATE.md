# دليل تحديث واجهة إدارة المستخدمين

لتقليل البيانات المنقولة في جدول إدارة المستخدمين مع الحفاظ على تفاصيل المستخدم عند الحاجة، تم تعديل نقاط النهاية التالية:

## جدول المستخدمين
- **المسار:** `GET /api/UsersForGroups/GetUsersForSelects`
- **المعاملات الجديدة:**
  - `includeRelations` (اختياري، افتراضي = `false`):
    - `false` → يعيد بيانات أساسية فقط للجدول (المعرف، الاسم الكامل، البريد الإلكتروني، رقم الهاتف، الدور... بدون العلاقات).
    - `true`  → يعيد نفس البيانات مع العلاقات (كما كان السلوك السابق).
- **المعاملات الحالية** (`UserTypeId`, `managerId`, `teacherId`, `branchId`, `nationalityId`, مع فلترة وترقيم الصفحات) تعمل كما هي.

**ما يحتاجه الفرونت:**
- عند تحميل جدول المستخدمين استخدم الاستدعاء بدون `includeRelations` (أو `includeRelations=false`) للحصول على البيانات الخفيفة.
- فقط في الحالات التي تحتاج فيها إلى نفس استجابة الجدول مع العلاقات (إن وجدت) استخدم `includeRelations=true`.

## تفاصيل المستخدم
- **المسار:** `GET /api/UsersForGroups/GetUserDetails`
- **المعاملات:**
  - `id` (إجباري): معرف المستخدم المطلوب.
- **الاستجابة:** تفاصيل المستخدم كاملة مع العلاقات التابعة له (مشرفين/معلمين/طلاب/... حسب نوع المستخدم).

**ما يحتاجه الفرونت:**
- عند الضغط على زر "عرض" لمستخدم ما، استبدل الاستدعاء الحالي باستدعاء `GetUserDetails?id=<USER_ID>` لجلب جميع البيانات المرتبطة.

## ملاحظات
- لم يتم تغيير أسماء الحقول في الاستجابة، فقط تم فصل وضع الجدول الخفيف عن وضع التفاصيل الكاملة.
- الترقيم (pagination) يبقى عبر `FilteredResultRequestDto` في نداء الجدول، بينما استدعاء التفاصيل يعيد عنصراً واحداً فقط.
